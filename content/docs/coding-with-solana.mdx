### JavaScript & Rust Basics

#### JavaScript Fundamentals

- **Variables**: `let`, `const`, `var`
- **Functions**: Arrow functions, async/await
- **Objects**: Destructuring, spread operator
- **Modules**: Import/export syntax
- **Error Handling**: Try-catch blocks

#### Rust Fundamentals

- **Ownership**: Unique concept for memory management
- **Borrowing**: References and borrowing rules
- **Lifetimes**: How long references are valid
- **Pattern Matching**: Match expressions and destructuring
- **Error Handling**: Result and Option types

#### Key Differences

- **Memory Management**: Manual vs automatic
- **Type System**: Static vs dynamic typing
- **Concurrency**: Ownership-based safety vs event loop
- **Performance**: Zero-cost abstractions vs JIT compilation

### JavaScript Queries for PDA Data

#### Setting Up Web3.js

```bash
npm install @solana/web3.js
```

#### Basic Connection

```javascript
import { Connection, PublicKey } from "@solana/web3.js";

const connection = new Connection("http://localhost:8899", "confirmed");
```

#### Querying Account Data

```javascript
// Get account info
const accountInfo = await connection.getAccountInfo(publicKey);

// Get multiple accounts
const accounts = await connection.getMultipleAccountsInfo(publicKeys);

// Get program accounts
const programAccounts = await connection.getProgramAccounts(programId);
```

#### PDA Queries

```javascript
import { PublicKey } from "@solana/web3.js";

// Find PDA
const [pda, bump] = PublicKey.findProgramAddress(
  [Buffer.from("seed"), owner.toBuffer()],
  programId
);

// Get PDA data
const pdaInfo = await connection.getAccountInfo(pda);
```

### Writing Basic Anchor Programs

#### Anchor Framework Overview

- **Macros**: Simplify common program patterns
- **Accounts**: Type-safe account validation
- **Instructions**: Structured program logic
- **Errors**: Custom error handling
- **Testing**: Built-in testing framework

#### Program Structure

```rust
use anchor_lang::prelude::*;

declare_id!("your_program_id");

#[program]
pub mod your_program {
    use super::*;

    pub fn initialize(ctx: Context<Initialize>) -> Result<()> {
        // Your logic here
        Ok(())
    }
}

#[derive(Accounts)]
pub struct Initialize {
    #[account(init, payer = user, space = 16 + 16)]
    pub base_account: Account<'info, BaseAccount>,
    #[account(mut)]
    pub user: Signer<'info>,
    pub system_program: Program<'info, System>,
}

#[account]
pub struct BaseAccount {
    pub count: u64,
}
```

#### Key Concepts

- **Accounts**: Data storage on-chain
- **Context**: Account validation and constraints
- **Instructions**: Public functions that can be called
- **Seeds**: Deterministic account derivation
- **Space**: Account storage allocation

### Git Basics

#### Version Control Fundamentals

- **Repository**: Project history and version tracking
- **Commit**: Snapshot of project at a point in time
- **Branch**: Independent line of development
- **Merge**: Combining changes from different branches
- **Remote**: Shared repository (GitHub, GitLab)

#### Essential Commands

```bash
# Initialize repository
git init

# Add files to staging
git add .

# Commit changes
git commit -m "Initial commit"

# Create and switch branches
git checkout -b feature-branch

# Push to remote
git push origin main
```

#### Best Practices

- **Commit Messages**: Clear, descriptive messages
- **Branch Naming**: Feature/issue-based naming
- **Regular Commits**: Small, logical changes
- **Pull Requests**: Code review before merging
